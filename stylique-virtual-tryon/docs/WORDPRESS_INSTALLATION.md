# WordPress / WooCommerce Plugin Installation Guide

This guide covers installing and configuring the Stylique Virtual Try-On plugin on a WordPress + WooCommerce site.

---

## Prerequisites

- WordPress 5.0+
- WooCommerce plugin installed and activated. The Stylique plugin will not load without it.
- A running Stylique backend with a public HTTPS URL.
- Supabase migration `008_woocommerce_connect_fields.sql` applied before using the WooCommerce connect flow.

Project delivery note: the plugin ships with `https://stylique-api.onrender.com` as the packaged backend default because it is the current test Render API. Before handing this repo to a client or Stylique owner, replace that default in the plugin or ask them to paste their own backend URL in **Settings -> Stylique Try-On**.

---

## Step 1: Install the Plugin

### Option A: Upload via WordPress Admin

1. Download the plugin folder `wordpress-stylique-virtual-tryon` from the project repository.
2. Compress the inner `wordpress-stylique-virtual-tryon` folder into a `.zip` file.
3. In WordPress Admin, go to **Plugins -> Add New -> Upload Plugin**.
4. Choose the `.zip` file and click **Install Now**.
5. Click **Activate Plugin**.

### Option B: Manual Upload via FTP/File Manager

1. Upload the `wordpress-stylique-virtual-tryon` folder to `wp-content/plugins/`.
2. In WordPress Admin, go to **Plugins -> Installed Plugins**.
3. Find **Stylique Virtual Try-On** and click **Activate**.

---

## Step 2: Connect Stylique

1. Go to **Settings -> Stylique Try-On** in the WordPress admin menu.
2. Confirm the **Backend API URL**.
3. Click **Connect Stylique**.
4. After connection succeeds, WordPress automatically saves the Store ID, Backend API URL, and Sync Secret.
5. Copy the one-time Store Panel password shown in the success notice.

The connect flow calls:

```http
POST {backend_url}/api/woocommerce/connect
```

The backend creates or updates the Supabase store, stores only hashed credentials, and returns the plain Store Panel password and sync secret only once.

---

## Step 3: Optional Manual Settings

The Connect button should fill these automatically. Use manual settings only for debugging or migration.

| Setting | Description | Example |
|---------|-------------|---------|
| **Store ID** | The connected WooCommerce domain stored in Supabase. | `example.com` |
| **Backend API URL** | The public HTTPS URL of your Stylique backend, with no trailing slash. | `https://stylique-api.onrender.com` |
| **Sync Secret** | Generated by Connect Stylique and sent as `X-Stylique-Sync-Secret` during product sync. | Auto-generated |
| **Primary Color** | Brand color for the widget UI. | `#642FD7` |
| **Secondary Color** | Accent color for the widget UI. | `#F4536F` |

Click **Save Changes** only if you edit these manually.

## Step 4: Verify Product Sync

Once the plugin is active and connected, products sync automatically when you save or update them in WooCommerce.

### Trigger a Full Sync

Use the built-in bulk sync button for old products:

1. Go to **Settings -> Stylique Try-On**.
2. In **Sync Existing Products**, click **Sync All Products**.
3. Keep the page open while products are processed in batches of 20.
4. Review the synced, failed, and total counts shown in the progress message.

Each published product will be POSTed to `{backend_url}/api/sync/woocommerce` with the generated sync secret. New and updated products still auto-sync when saved.

### Verify Sync Worked

1. **WordPress admin notices**: after saving a product, a success/failure notice appears briefly at the top of the editor.
2. **Backend logs**: look for WooCommerce sync logs.
3. **Supabase**: check the `inventory` table for rows with matching `woocommerce_product_id`.

### What Gets Synced

| WooCommerce Field | Inventory Column |
|-------------------|------------------|
| Product title | `product_name` |
| Description | `description` |
| Price | `price` |
| Permalink | `product_link` |
| Featured image + gallery | `image_url`, `images`, and image processing |
| Variations / attributes | `sizes` |
| Post ID | `woocommerce_product_id` |

## Step 5: Test the Widget

1. Navigate to any single product page on your store's frontend.
2. The Stylique widget appears below the Add to Cart button.
3. Test the login form, product availability check, photo upload, 2D try-on, and size recommendation.

---

## How It Works

### Widget Placement

The widget is automatically injected on single WooCommerce product pages using the `woocommerce_after_add_to_cart_form` hook. No shortcode or block is needed.

### Frontend API Calls

The widget JavaScript communicates with the Stylique backend via:

- `/plugin/auth`
- `/plugin/check-product`
- `/plugin/store-status`
- `/plugin/embed-tryon-2d`
- `/plugin/size-recommendation`
- `/plugin/track-conversion`

### Product Sync Triggers

The plugin hooks into multiple WooCommerce events to ensure products sync reliably:

- `woocommerce_new_product`
- `woocommerce_update_product`
- `woocommerce_product_object_updated_props`
- `save_post_product`
- `wp_insert_post`

Sync is deferred to the `shutdown` hook at priority 99 during admin product saves so WooCommerce metadata, including gallery images, is saved before the API call.

## Troubleshooting

### Connect Stylique fails

1. Confirm the backend URL is public HTTPS and has no trailing slash.
2. Confirm the backend has been deployed with the latest WooCommerce connect route.
3. Confirm Supabase migration `008_woocommerce_connect_fields.sql` has been applied.
4. Check Render/backend logs for `[WooCommerce Connect]`.

### Widget does not appear

1. Confirm WooCommerce is active.
2. Confirm you are viewing a single product page.
3. Check whether the theme supports the `woocommerce_after_add_to_cart_form` hook.
4. Open the browser console and look for JavaScript errors.

### Products are not syncing

1. Click **Connect Stylique** first so Store ID and Sync Secret are generated.
2. Confirm the product is published.
3. Confirm the backend is reachable from the WordPress server.
4. Check the WordPress admin notice after saving a product.
5. Check backend logs for WooCommerce sync errors.

### Sync timeout

The plugin has a 15-second timeout for product sync and a 20-second timeout for connect. If the backend is cold-starting, retry the action.

## Settings Reference

| Option Key | Description | Default |
|------------|-------------|---------|
| `stylique_store_id` | Connected store identifier. | Empty until connect |
| `stylique_backend_url` | Backend API URL. | `https://stylique-api.onrender.com` |
| `stylique_sync_secret` | Per-store WooCommerce sync secret generated during connect. | Empty until connect |
| `stylique_primary_color` | Primary brand color. | `#642FD7` |
